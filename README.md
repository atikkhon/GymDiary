# GymDiary

**GymDiary** - это приложение на **C++ / Qt Quick / QML** для ведения личного дневника тренировок. Проект позволяет быстро фиксировать выполненные подходы: выбрать упражнение, дополнительный вес и количество повторений, после чего запись автоматически добавляется в лог тренировки.

**Цель проекта** - упростить ведение дневника тренировок, который является важной частью контроля личного прогресса. Приложение делает акцент на простом интерфейсе и минимальном количестве действий: пользователь выбирает нужные параметры и сразу сохраняет подход без ручного ввода длинных записей.

## Что делает приложение

**GymDiary** помогает фиксировать тренировочные подходы и просматривать историю тренировок в удобном виде. Пользователь выбирает упражнение, вес и количество повторений, а приложение сохраняет эту информацию в локальный JSON-файл.

Основные возможности:

- выбор вида упражнения;
- выбор дополнительного веса;
- выбор количества повторений;
- автоматическое добавление подхода после выбора повторений;
- просмотр лога выполненных подходов;
- удаление отдельной записи из лога подходов;
- группировка данных по тренировочным дням;
- просмотр дневника тренировок на отдельном экране;
- переход к дневнику тренировок через свайп влево;
- локальное хранение данных в JSON;
- разделение логики данных на C++ backend и интерфейса на QML.

## Архитектура

Проект построен по схеме **C++ backend + QML frontend**.

C++-часть отвечает за хранение данных, работу с JSON, модели для отображения данных и передачу информации в QML. QML-часть отвечает за пользовательский интерфейс, выбор упражнений, веса, повторений и отображение тренировочного лога.

| Файл | Роль |
|---|---|
| `main.cpp` | Запускает Qt-приложение, устанавливает стиль `Fusion`, регистрирует C++ классы `JsonDataBase`, `DiaryModel`, `SetsLogModel` и `RepsModel` для использования в QML. |
| `domain.h` | Описывает основные структуры предметной области: `Set`, `Exercise`, `TrainingDay`. Эти структуры отражают логику хранения тренировок: день -> упражнения-> подходы. |
| `jsondatabase.h / jsondatabase.cpp` | Основная логика работы с данными: создание JSON-файла, чтение тренировок, запись новых подходов, удаление подходов и отправка сигнала `dataChanged` при изменении данных. |
| `repsmodel.h / repsmodel.cpp` | Модель выбора значений для интерфейса: список упражнений, список весов и список повторений. Также реализует переключение видимых элементов через кнопки влево/вправо. |

| `diarymodel.h / diarymodel.cpp` | Модель дневника тренировок на базе `QAbstractListModel`. Группирует подходы по дате и упражнению для отображения истории тренировок. |
| `main.qml` | Основной QML-файл приложения. Создает экземпляры C++ моделей и базы данных, а также содержит `SwipeView` для переключения между главным экраном и дневником. |
| `MainScreen.qml` | Главный экран приложения: выбор упражнения, веса и повторений, а также отображение текущего лога подходов. |
| `SetsLog.qml` | Компонент отображения списка выполненных подходов. Использует `ListView`; выбранную запись можно удалить двойным нажатием. |
| `TableViewScreen.qml` | Экран дневника тренировок. Отображает историю тренировок через `ListView` с группировкой по дате. |
| `LabelAndOptions.qml`, `LeftButton.qml`, `RightButton.qml`, `MyButton.qml`, `FourButtons.qml` | Переиспользуемые QML-компоненты для построения интерфейса выбора упражнений, веса и повторений. |

## Как реализовано

### Хранение данных

Данные тренировок хранятся локально в формате **JSON**. При первом запуске приложение проверяет наличие файла `GymDiary.json` в директории приложения. Если файла нет, он создается из шаблона, который подключен через Qt resources.

Структура данных построена вокруг трех сущностей:

- **TrainingDay** - тренировочный день с датой;
- **Exercise** - упражнение внутри конкретного дня;
- **Set** - отдельный подход с весом, количеством повторений и уникальным `id`.

После добавления или удаления подхода C++ backend обновляет JSON-файл и отправляет сигнал `dataChanged`. Модели, подключенные к базе данных, получают этот сигнал и перестраивают данные для интерфейса.

### Model-View-Delegate подход

В проекте применяется подход **Model-View-Delegate**, который хорошо подходит для Qt/QML-приложений.

- `JsonDataBase` хранит и изменяет данные;
- `SetsLogModel` подготавливает данные для лога подходов;
- `DiaryModel` подготавливает данные для дневника тренировок;
- `RepsModel` подготавливает значения для выбора упражнений, весов и повторений;
- `ListView` отображает данные моделей в QML;
- QML-делегаты отвечают за внешний вид каждой строки в списке.

Такое разделение позволяет не смешивать интерфейс, отображение данных и логику работы с файлом.

### Логика добавления подхода

На главном экране пользователь последовательно выбирает:

1. упражнение;
2. дополнительный вес;
3. количество повторений.

После выбора повторений приложение вызывает метод `addSet(...)` у C++ объекта базы данных. Новый подход добавляется к текущей дате. Если для текущей даты еще нет записи, создается новый тренировочный день. Если для выбранного упражнения еще нет записи внутри этого дня, создается новое упражнение.

### Логика удаления подхода

Каждый подход имеет уникальный `id`. При удалении приложение ищет подход с нужным `id` в JSON-структуре и удаляет его. Если после удаления у упражнения не осталось подходов, упражнение удаляется из тренировочного дня. Если после этого в тренировочном дне не осталось упражнений, удаляется и сам тренировочный день.

## Как запустить

1. Скачать APK-файл из релиза проекта.
2. Установить приложение на Android-устройство.
3. При необходимости разрешить установку приложения из внешнего источника.
4. Запустить **GymDiary**.

## Какой опыт получен

Были получены практические навыки при разработке проекта, а именно:

- разработка приложения на **C++ / Qt Quick / QML**;
- интеграция C++ классов с QML через `qmlRegisterType`;
- использование `Q_PROPERTY` для передачи данных и моделей в QML;
- работа с `QAbstractListModel` и собственными ролями данных;
- обновление QML-интерфейса при изменении C++ данных;
- использование Qt signals/slots для синхронизации базы данных и моделей;
- чтение и запись JSON через `QJsonDocument`, `QJsonObject` и `QJsonArray`;
- хранение пользовательских данных через `QStandardPaths::AppDataLocation`;
- построение интерфейса на `ListView`, `Repeater`, `SwipeView` и переиспользуемых QML-компонентах;
- разделение приложения на backend, модели данных и UI-компоненты;
- сборка приложения под Android.
